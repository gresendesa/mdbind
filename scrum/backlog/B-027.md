## Description

Reescrever o `README.md` para ser a página inicial ideal do projeto no GitHub.
O objetivo é torná-lo visualmente chamativo, elegante e informativo para quem
chega ao repositório pela primeira vez — seja um desenvolvedor, um potencial
contribuidor ou um usuário de agentes de IA.

## Motivação

O README atual é funcional, mas não foi concebido como vitrine do projeto no
GitHub. Falta:

- Headline impactante e posicionamento claro
- Badges (CI, versão, licença, Python)
- Visual hierarchy com seções bem delimitadas
- GIF ou screenshot de uso no terminal
- Quick-start enxuto (≤ 5 linhas)
- Tabela de comandos resumida e scannable
- Links de contribuição / licença no rodapé

## Escopo

### Estrutura proposta do novo README

| Seção | Conteúdo |
|-------|----------|
| Hero | Título, tagline de 1 linha, badges |
| Motivação | Por quê MdBind? (3 bullets) |
| Instalação rápida | `pip install` / `git clone` em 3 passos |
| Uso rápido | 3-5 comandos com saída esperada |
| Tabela de comandos | Todos os comandos com descrição curta |
| Conceitos core | Seção, diretivas `@include`/`@ref`, URI |
| Filosofia | 5 princípios (do atual README) |
| Exemplos | Link para `examples/` |
| Contributing / License | Seção final |

### Requisitos visuais

- Usar badges shields.io: `Python`, `License`, `Tests`
- Usar blocos de código com syntax highlighting correto
- Evitar parágrafos longos — preferir bullets e tabelas
- Heading hierarchy clara (H1 → H2 → H3)
- Compatível com renderização GitHub (sem extensões externas)

### O que NÃO mudar

- Conteúdo técnico do specification.md (não é escopo)
- Arquivos `examples/`
- Nenhuma funcionalidade do código
