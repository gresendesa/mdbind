## Anticipated blockers and mitigations

- Blocker: existing parser may not retain exact source ranges for structured
  YAML blocks.
  Mitigation: inspect parser boundaries first and add a narrow rewrite helper
  with regression tests proving body preservation.
- Blocker: dotted paths can conflict with scalar intermediate values.
  Mitigation: return a clear error when an intermediate value is not a mapping,
  as required by B-032.
- Blocker: removing metadata can accidentally remove sibling values if path
  handling is imprecise.
  Mitigation: isolate unset behavior in helper tests for top-level and nested
  paths before exposing it through the CLI.
- Blocker: YAML serialization may alter metadata formatting more broadly than
  intended.
  Mitigation: restrict rewrite to the structured block and test that body and
  directives outside it remain unchanged.
- Blocker: command family contract can drift across `get`, `update`, and
  `unset`.
  Mitigation: define shared section id, dotted path, and error semantics before
  implementing the subcommands.
