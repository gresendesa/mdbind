# B-049 - Support language selection in mdb init

- Status: done
- PO Priority: 2 (high)
- Risk: 2 (medium)
- Owner: gresendesa
- Creation date: 2026-06-14
- Last updated: 2026-06-14
- Dependencies: None

## Description

Allow the user to select the documentation language when initializing a workspace using `mdb init`.
The supported languages for now are:
- English (`en`)
- Portuguese (`pt_br`)

Templates will be restructured to organize documentation files into language-specific folders (e.g., `en/` and `pt_br/`). During initialization, the engine will only copy and render the template files belonging to the chosen language.

## Out of scope

- Translation of configuration files or templates themselves (only the documentation content folders inside the templates are filtered by language).
- Languages other than `en` and `pt_br`.

## Criteria of acceptance

- [x] `mdb init` supports a `--lang` option (e.g., `--lang en` or `--lang pt_br`).
- [x] If `--lang` is omitted and stdin is a TTY, the CLI interactively prompts the user to select the language.
- [x] Templates contain separated subfolders for `en` and `pt_br` content.
- [x] Initialization only extracts, renders, and places the documents of the selected language at the destination.
- [x] The generated workspace successfully passes `mdb validate`.
- [x] Automated tests cover both command-line option and interactive prompting.

## Definition of Done

- [x] Technical planning completed with PO Priority assigned.
- [x] Implementation completed.
- [x] Automated tests executed successfully.
- [x] Regression checklist executed and documented.
- [x] Project memory updated under `scrum/`.

## Risk assessment
