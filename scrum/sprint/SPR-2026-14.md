## Anticipated blockers and mitigations

- Blocker: JSON Schema validation may require a new runtime dependency.
  Mitigation: inspect dependency options first and keep the supported schema
  draft documented.
- Blocker: schema error fields can break existing `validate --json` consumers if
  the base shape changes.
  Mitigation: preserve existing `type`, `uri`, and `detail` fields and add
  schema-specific fields only additively.
- Blocker: schema references could accidentally introduce global schema
  semantics.
  Mitigation: validate only sections that explicitly declare `schema`.
- Blocker: web URI schemas could introduce nondeterminism.
  Mitigation: B-033 must not resolve web URIs; it should produce a documented
  deterministic unsupported/validation error and leave remote resolution to
  B-034.
- Blocker: local schema path resolution can be ambiguous.
  Mitigation: document resolution relative to the repository root, with
  `scrum/schema/` as the recommended location.
