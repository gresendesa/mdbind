## Description

Implement automated environment discovery and system-instruction hooking immediately after or during `mdb init`. This process automatically injects hooks pointing to the project's `CONSTITUTION.md` and rules inside the target development environment's entrypoint instruction file. 

Supported environments & entrypoints:
- **Antigravity / Codex**: `AGENTS.md` at the workspace root.
- **GitHub Copilot**: `.github/copilot-instructions.md`.
- **Custom / Others**: configurable templates.

The hook must be injected using MdBind directive notation (e.g., `@include: scrum/CONSTITUTION.md`) so that `mdb` can verify programmatically if the hook was successfully installed.
Additionally, provide tools/instructions to verify the "awareness" (health check) of the LLM/agent with respect to these rules, such as a canary phrase trigger where the LLM must respond with a constitutionally mandated output.

## Scope

- Automatically detect the active development environment (inspecting workspace files, environment variables, or config).
- Inject/append the MdBind hook pointing to `CONSTITUTION.md` in the discovered agent instruction files (e.g. `AGENTS.md`, `.github/copilot-instructions.md`).
- Implement an `mdb health` or `mdb verify-awareness` command to:
  - Check if the instructions files contain the required MdBind hooks.
  - Suggest a canary test prompt to verify if the LLM/agent session is successfully restricted by the rules.
- Print clear instructions asking the user/agent to restart the session if a hook is newly inserted.

## Out of scope

- Direct API injection into proprietary IDE memory spaces (we operate strictly through workspace files).

## Criteria of acceptance

- [ ] `mdb init` includes options or automatic detection to write instruction hooks in `AGENTS.md` and `.github/copilot-instructions.md`.
- [ ] Hooks are written using standard MdBind syntax, allowing `mdb validate` or a new command to check their presence.
- [ ] User is guided to restart the session with clear terminal outputs.
- [ ] A diagnostic/canary mechanism exists to verify the agent's conformance to the constitution.
