## Description

Several CLI command docstrings in `src/mdbind/cli.py` are still written in Portuguese.
The CONSTITUTION (§0) mandates English as the official language for all code, documentation,
and comments. The `mdb --help` output must be fully in English to be consistent with the
rest of the project.

## Affected commands

| Command | Current (PT) | Expected (EN) |
|---|---|---|
| `get` | `Extrai uma secao com 100%% de fidelidade documental (linhas brutas do arquivo fonte).` | `Extracts a section with 100% documentary fidelity (raw source lines).` |
| `tree` | `Exibe a hierarquia visual de dependencias de uma secao.` | `Displays the visual dependency hierarchy of a section.` |
| `compose` | `Materializa um documento Markdown unificado expandindo @include recursivamente.` | `Materializes a unified Markdown document by recursively expanding @include directives.` |
| `validate` | `Verifica a integridade estrutural do repositorio de grafos Markdown.` | `Checks structural integrity of the Markdown graph repository.` |

`context`, `backlinks`, `search`, `impact`, `neighbors`, `explain`, `diff`, `query`,
and `context-compose` are already in English — no change needed.

## Scope

- `src/mdbind/cli.py`: translate docstrings for `get`, `tree`, `compose`, `validate`
- Any inline Portuguese comments in `cli.py` should also be translated
- No changes to logic, signatures, or test files

## Criteria of acceptance

- [ ] `mdb --help` output is fully in English
