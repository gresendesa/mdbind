# B-034 - Web URI schema references for section metadata

- Status: todo

## Description

Extend section metadata schema validation to support schemas referenced by web
URI values in the `schema` metadata attribute.

This item is intentionally separate from B-033 so local schema validation can
remain deterministic and local-first. Remote schema resolution should be added
only after the local validation contract is stable.

## Scope

- Support `schema` values that point to web URIs.
- Define deterministic error behavior for unavailable network access, failed
  downloads, non-2xx responses, invalid content types, invalid schema documents,
  and timeouts.
- Decide whether remote schemas are fetched every run, cached, pinned by hash,
  or explicitly not cached.
- Preserve B-033 local schema behavior.
- Preserve `mdb validate --json` backward compatibility through additive
  fields.
- Document remote schema security and reproducibility trade-offs.

## Out of scope

- Replacing local schema validation.
- Requiring network access for repositories that use only local schemas.
- Defining a global repository schema.

## Criteria of acceptance

- [ ] `mdb validate` can resolve a schema from a web URI when network access is
      available.
- [ ] Network failures produce deterministic validation errors.
- [ ] Invalid remote schema documents produce deterministic validation errors.
- [ ] Remote schema errors include section URI, schema URI, and detail.
- [ ] Local schema validation from B-033 remains unchanged.
- [ ] Automated tests cover successful remote schema resolution through a
      controlled test server or mock, unavailable remote schema, invalid remote
      schema, and local schema regression.
- [ ] Documentation explains remote schema reproducibility and security risks.

## Definition of Done

- [ ] Technical planning completed with PO Priority assigned.
- [ ] Implementation completed.
- [ ] Automated tests executed successfully.
- [ ] Documentation updated.
- [ ] Manual test documented.
- [ ] Regression checklist executed and documented.
- [ ] Project memory updated under `scrum/`.

## Dependencies

- B-033 must first define local per-section schema validation and shared
  validation error contracts.

## Risk assessment

- Implementation risk: medium, because remote retrieval and failure handling
  must be deterministic.
- Contract risk: medium, because remote schema references introduce
  reproducibility and caching decisions.
- Security risk: medium, because fetching remote schemas can expose users to
  untrusted content and availability issues.

## Notes

- The PO explicitly requested this as a later, low-priority item.
- Local-first schema validation remains the preferred default.
