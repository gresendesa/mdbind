## Description

Implement automated environment discovery and system-instruction hooking immediately after or during `mdb init`. This process automatically injects hooks pointing to the project's `CONSTITUTION.md` and rules inside the target development environment's entrypoint instruction file.

Supported environments & entrypoints:
- **Antigravity / Codex**: `AGENTS.md` at the workspace root.
- **GitHub Copilot**: `.github/copilot-instructions.md`.
- **Custom / Others**: configurable templates.

### Non-negotiable Rules:
1. **Respect Existing Entrypoint Files**: If the entrypoint file already exists, it must **never** be overwritten. Instead, the hook must be respectfully inserted around the existing content.
2. **Interactive Hook Placement**: The user must be asked (via CLI prompts or configurable options) whether the hook should be prepended (inserted at the beginning) or appended (inserted at the end) of the existing file.
3. **Secret Verification Phrase**: The user will have a unique secret phrase generated automatically during workspace initialization and saved to `.mdb/config.yaml`. This phrase **must consist of exactly five random English words separated by spaces** (e.g., `whispering forest blue anchor flight`). When this phrase is input to the LLM at any point in the session, the LLM must provide a standard response summarizing the key points of the installed/modified template documentation that it is aware of.

The hook must be injected using MdBind directive notation (e.g., `@include: scrum/CONSTITUTION.md`) so that `mdb` can verify programmatically if the hook was successfully installed.
Additionally, provide tools/instructions to verify the "awareness" (health check) of the LLM/agent with respect to these rules.

## Scope

- Automatically detect the active development environment (inspecting workspace files, environment variables, or config).
- Prompt the user to choose whether to insert the hook at the beginning or the end of the file.
- Insert the MdBind hook pointing to `CONSTITUTION.md` (e.g., `@include: scrum/CONSTITUTION.md`) in the target agent instruction files (e.g. `AGENTS.md`, `.github/copilot-instructions.md`) without overwriting existing content.
- Implement an automatic 5-word passphrase generator using a wordlist inside `mdb` during `mdb init`, and save it under `.mdb/config.yaml`.
- Write instructions into the target entrypoint files explaining the secret phrase trigger and how the LLM should respond.
- Implement the `mdb check-session-hook` command to:
  - Verify if the agent instruction entrypoint files exist and contain the required MdBind hooks pointing to `CONSTITUTION.md`.
  - Display the generated 5-word secret verification phrase for the user to copy/paste into their agent session.
- Print clear instructions asking the user/agent to restart the session if a hook is newly inserted.

## Out of scope

- Direct API injection into proprietary IDE memory spaces (we operate strictly through workspace files).

## Criteria of acceptance

- [x] `mdb init` detects existing agent entrypoint files and inserts the hook without overwriting them.
- [x] `mdb init` prompts the user to place the hook at either the beginning or the end of the entrypoint file.
- [x] A unique secret phrase is generated/defined in `.mdb/config.yaml`.
- [x] Entrypoint files instruct the LLM to output a standard summary of key template points upon receiving the secret phrase.
- [x] Hooks are written using standard MdBind syntax, allowing `mdb check-session-hook` to verify their presence.
- [x] User is guided to restart the session with clear terminal outputs.

## Notes

- Critical to ensure that AI agents initializing or entering a workspace are immediately bound by the local repository's constitution.
- The random 5-word secret phrase is key to verification: it prevents LLMs from reacting using their parametric memory (pre-trained weights) once `mdbind` becomes famous. By forcing a dynamic, randomized phrase, we guarantee the LLM has actually read the current workspace's instruction files and is not hallucinating or replying based on static pre-trained knowledge.
