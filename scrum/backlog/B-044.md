## Description

Provide a dedicated `mdb session-hook` command group to allow developers to inject, update, or remove the MdBind session rules hooks in development environment instruction files at any time. This enables developers to change or swap IDEs/environments multiple times without re-initializing the workspace.

Actions needed:
- `mdb session-hook inject`: Prompt or accept path to inject/update the hook.
- `mdb session-hook remove`: Locate and strip the hook from the file.

## Out of scope

- Parsing non-standard markdown comments for hook boundary identification.

## Criteria of acceptance

- [x] Command `mdb session-hook inject` injects/updates the hook in a specified or auto-detected entrypoint file.
- [x] Command `mdb session-hook remove` removes the hook from a specified or auto-detected entrypoint file.
- [x] Hook removal is clean, leaving other file contents completely intact.
- [x] Both commands accept optional `--file <path>` to target a custom entrypoint (e.g. `.cursorrules`).
- [x] Config file `.mdb/config.yaml` is updated on injection/removal to keep track of active hooks.
