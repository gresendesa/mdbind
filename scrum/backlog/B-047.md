# B-047 - Interactive template selection in mdb init when template is omitted

- Status: done
- PO Priority: 2 (high)

- Owner: gresendesa
- Creation date: 2026-06-14
- Last updated: 2026-06-14
- Dependencies: None

## Description

Provide a user-friendly fallback in `mdb init` when the template option (`--template` / `-t`) is omitted. Instead of failing or requiring a template parameter:
1. `mdb init` will dynamically list the available templates in the templates directory.
2. Prompt the user to select one.
3. Automatically pack the chosen template in the background.
4. Proceed with the normal initialization workflow.

## Out of scope

- Downloading remote templates dynamically during listing (only local templates from the `templates/` folder or built-in list are scanned).

## Criteria of acceptance

- [x] Executing `mdb init` without `-t` or `--template` does not raise an error.
- [x] If stdin is a TTY, the user is prompted with a list of available templates.
- [x] The templates list is fetched dynamically (e.g. scanning the `templates/` package directory).
- [x] Selecting a template automatically packs it in a temporary file and runs standard `init_from_template_package`.
- [x] Automated tests verify the interactive selection fallback.
