# B-035 - File-scoped validation mode for `mdb validate`

## Description

Add a file-scoped validation mode to `mdb validate` so users can validate a
single Markdown file without validating every Markdown file under a repository
root.

The current `mdb validate --root <dir>` behavior remains the integrated
repository validation mode. The new mode should support:

```bash
mdb validate --file path/to/file.md
```

This is intended for manual and iterative editing workflows where the user wants
fast feedback for one file and does not want unrelated repository fixtures,
drafts, or historical files to affect the result.

## Scope

- Add `--file <path.md>` to `mdb validate`.
- Keep `--root <dir>` available and unchanged.
- Reject simultaneous use of `--root` and `--file` with a clear CLI error.
- Validate only the selected file when `--file` is used.
- Preserve existing `mdb validate --json` output shape.
- Preserve local schema validation behavior from B-033 for sections in the
  selected file.
- Document that `--file` is isolated validation. References or includes to
  sections outside the selected file may be reported as broken unless future
  behavior explicitly loads an external context.
- Update `README.md`, `specification.md`, and `llms.txt`.

## Out of scope

- Section-scoped validation by URI, such as `mdb validate file.md#section`.
- Loading a full repository graph while reporting only one file.
- Changing the default `mdb validate` root behavior.
- Changing schema resolution semantics from B-033.

## Criteria of acceptance

- [ ] `mdb validate --file path/to/file.md` validates only that Markdown file.
- [ ] `mdb validate --root docs/` keeps current recursive root behavior.
- [ ] Passing both `--root` and `--file` fails with a clear error.
- [ ] `--file` works with `--json`.
- [ ] `--file` validates local per-section schemas declared in the selected
      file.
- [ ] Tests cover successful file validation, file-local schema failure,
      `--root` plus `--file` conflict, missing file, and JSON output.
- [ ] Documentation explains when to use `--root` versus `--file`.

## Risk assessment

- Implementation risk: medium, because the current validation flow indexes a
  root and must gain a file-limited path without breaking existing behavior.
- Contract risk: low, because this is an additive CLI option.
- Semantics risk: medium, because isolated validation can report cross-file
  references as broken; this must be documented clearly.
