## Description

Add four new template packages (`kanban`, `product`, `engineering`, `minimal`) to the `templates/` directory to support a wider range of software development and project management methodologies during workspace initialization with `mdb init`.

Templates to implement:
1. **Kanban**: A flow-based template featuring `BOARD.md`, a `backlog/` folder, continuous decisions and lessons learned logs, and custom agent rules.
2. **Shape Up (Product-Driven)**: A product-centric template featuring `PITCHES.md`, `cycles/` and `specs/` directories, a product decisions log, and custom agent rules.
3. **ADR/RFC-Centric**: An engineering-focused template featuring `ADR.md` (index of ADRs), `adr/` and `rfcs/` directories, a high-level `ROADMAP.md`, and custom agent rules.
4. **Minimal**: An ultra-lightweight template featuring a minimal `README.md`, a single `todo.md` file, a basic `CHANGELOG.md`, and basic agent rules.

Actions needed:
- Create source files for each template under `templates/<name>/`.
- Define `manifest.yaml` for each template with appropriate variables, mappings, and entrypoints.
- Update `src/mdbind/template_packages.py` or packaging scripts if they hardcode default templates or profiles.
- Generate deterministic `SIGNATURE.yaml` checksum signatures for all new packages so they can be bundled or initialized safely.
- Write unit/integration tests to verify that `mdb init` can instantiate each of these template packages successfully.

## Out of scope

- Setting up remote repositories or remote hosting for these templates (they should be packaged locally as zip files or directories ready for packaging).
