## Description

Verify that a template workspace (or repository) complies with the minimum template conformity rules:
1. Every template must contain a `CONSTITUTION.md` file at the root.
2. All `.md` files in the template must be reachable (directly or indirectly) from `CONSTITUTION.md` via `@ref` or `@include` directives.

This check must be executed deterministically by `mdb validate` (when running in a template/workspace validation context, or generally when verifying repository integrity).

## Out of scope

- Automated fixing/modification of templates (read-only verification).

## Criteria of acceptance

- [x] Executing `mdb validate` detects if `CONSTITUTION.md` is missing at the repository root and reports it as an error.
- [x] Executing `mdb validate` detects any `.md` file in the workspace that is not reachable (directly or indirectly) from the `CONSTITUTION.md` root via `@ref` or `@include` directives and reports it as an error.
- [x] The checks run deterministically.
- [x] Verification can be run as part of the default `mdb validate` or via a dedicated option.
- [x] Automated tests cover this conformity check.
