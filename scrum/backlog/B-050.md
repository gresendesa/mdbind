# B-050 - Fix local template packaging inside Python package

- Status: done
- PO Priority: 1 (critical)
- Risk: 1 (low)
- Owner: gresendesa
- Creation date: 2026-06-14
- Last updated: 2026-06-14
- Dependencies: None

## Description

When `mdbind` is packaged and installed from PyPI/wheel (e.g. version 0.1.15), the `templates` directory located at the root of the repository is not included in the wheel package. This causes `mdb init` to fail with the error: `Error: No local templates found. Please specify a template package with --template.` when run in non-interactive/installed environments.

To fix this, we need to:
1. Move the `templates` folder under `src/mdbind/templates`.
2. Configure `pyproject.toml` and `MANIFEST.in` to ensure that `src/mdbind/templates` is packaged as package data and included in the wheel.
3. Update `locate_templates_dir()` in `src/mdbind/cli.py` to prioritize looking in the new package location.
4. Update automated tests in `tests/test_cli_init_pack.py` to resolve templates dynamically using `locate_templates_dir()`.

## Out of scope

- Changing any template rendering logic.
- External remote/web templates (already working).
